# Trend Table Viewer (React Native)

This app displays the table outputs generated by `trendatestocks_v1.1.py`.

## 1) Generate latest JSON from Python pipeline

From repo root:

```powershell
python .\trendatestocks_v1.1.py
```

This writes:

- `excel_output\trend_tables.json`

## 2) Install app dependencies

```powershell
cd .\mobile_table_viewer
npm.cmd install
```

## 3) Sync latest data into app assets

```powershell
npm.cmd run sync-data
```

## 4) Run on Android device/emulator

```powershell
npx.cmd expo run:android
```

## Build Android APK (EAS)

1. Install EAS CLI:

```powershell
npm.cmd install -g eas-cli
```

2. Login and configure:

```powershell
eas login
eas build:configure
```

3. Build APK:

```powershell
npm.cmd run build-apk
```

EAS returns a downloadable APK link when the build completes.

## Local release APK

This project also supports local Gradle release builds using `android/keystore.properties`.

```powershell
cd .\mobile_table_viewer\android
gradlew.bat assembleRelease -PreactNativeArchitectures=arm64-v8a
```

APK output:

- `mobile_table_viewer\android\app\build\outputs\apk\release\app-release.apk`

## Local emulator release APK

For the local `x86_64` Android emulator, use the dedicated build script:

```powershell
cd .\mobile_table_viewer
npm.cmd run release-apk-emulator
```

APK output:

- `mobile_table_viewer\android\app\build\outputs\apk\release\app-release-x86_64-emulator.apk`

Note:

- This build is signed with the release keystore, but targets only the `x86_64` emulator ABI.
- It is for emulator testing, not for installing on a real ARM64 phone.

## Local Play Store bundle

This is the recommended store-upload path on this machine:

```powershell
cd .\mobile_table_viewer
npm.cmd run bundle-release
```

Bundle output:

- `mobile_table_viewer\android\app\build\outputs\bundle\release\app-release.aab`

Note:

- The bundle is signed with the release keystore configured in `android\keystore.properties`.
- It is currently built with `-PreactNativeArchitectures=arm64-v8a` because Expo native CMake hits a Windows path-length failure for the full multi-ABI build from this repo location.

## App data source

The UI reads from:

- `mobile_table_viewer\assets\trend_tables.json`

If this file is stale, rerun `npm.cmd run sync-data`.
